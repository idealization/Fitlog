# 서비스 요구사항 정의서: AI 옷장 스타일 추천 앱

- 작성일: 2026-06-06
- 문서 상태: 초안
- 서비스명: Fitlog
- 대상 플랫폼: iOS, Android 우선. 이후 Web 확장 가능

## 1. 서비스 개요

사용자가 자신의 옷장이나 개별 의류 사진을 촬영하면, 앱이 사진 속 의류를 식별하고 일러스트 형태의 디지털 아이템으로 변환하여 저장한다. 저장된 의류는 카테고리, 색상, 소재, 계절감, 스타일 태그 등으로 데이터화된다.

앱은 사용자의 요청한 분위기, 일정, 날씨, 시간대, 최신 패션 트렌드, 보유 의류 데이터를 함께 고려하여 착장을 추천한다. 매일 아침 사용자가 지정한 시간에는 날씨를 반영한 오늘의 코디를 알림으로 제공한다.

## 2. 서비스 목표

- 사용자가 자신의 옷장을 손쉽게 디지털화할 수 있게 한다.
- 사진 기반 의류 인식과 일러스트 변환을 통해 시각적으로 보기 좋은 디지털 옷장을 제공한다.
- 최신 패션 트렌드와 개인 취향을 반영해 실제 보유 옷으로 입을 수 있는 코디를 추천한다.
- 날씨와 시간, 장소, 일정 맥락을 고려한 실용적인 데일리 스타일링을 제공한다.
- 사용자가 입력한 원하는 느낌을 자연어로 이해하여 맞춤형 스타일을 제안한다.

## 3. 핵심 사용자

### 3.1 1차 사용자

- 매일 코디를 고민하는 20-30대 사용자
- 옷은 많지만 어떤 조합이 좋은지 판단하기 어려운 사용자
- 본인의 옷장을 디지털로 관리하고 싶은 사용자
- 트렌드를 참고하고 싶지만 직접 찾아보는 데 시간을 쓰기 싫은 사용자

### 3.2 2차 사용자

- 미니멀 옷장, 캡슐 옷장, 여행 짐 꾸리기에 관심 있는 사용자
- 패션 커머스와 연계해 부족한 아이템을 추천받고 싶은 사용자
- 스타일 기록과 착용 빈도 분석을 원하는 사용자

## 4. 주요 사용자 시나리오

### 4.1 옷장 등록

1. 사용자가 옷장, 행거, 서랍, 또는 개별 옷 사진을 촬영한다.
2. 앱이 사진 속 의류를 자동으로 감지한다.
3. 사용자는 감지된 옷 영역을 확인하고 필요 시 수정한다.
4. 앱이 각 의류를 일러스트 이미지로 변환한다.
5. 앱이 의류별 속성 데이터를 자동 생성한다.
6. 사용자는 잘못 인식된 정보를 수정하고 저장한다.

### 4.2 원하는 느낌 기반 추천

1. 사용자가 "깔끔한 출근룩", "요즘 유행하는 빈티지한 느낌", "첫 데이트에 과하지 않은 스타일"처럼 자연어로 요청한다.
2. 앱이 요청 의도, 상황, 선호 스타일을 해석한다.
3. 앱이 보유 의류, 날씨, 트렌드, 착용 이력, 사용자 취향을 고려해 코디 후보를 생성한다.
4. 사용자는 추천 코디를 확인하고 마음에 드는 조합을 저장하거나 피드백한다.

### 4.3 매일 아침 날씨 기반 추천

1. 사용자가 추천 알림 시간과 기본 위치를 설정한다.
2. 앱이 매일 지정 시간 전에 최신 날씨 정보를 조회한다.
3. 앱이 기온, 강수, 습도, 체감온도, 미세먼지 등 외부 조건을 고려한다.
4. 앱이 오늘 입기 좋은 코디를 생성한다.
5. 앱이 지정 시간에 푸시 알림으로 추천 코디를 제공한다.

## 5. 기능 요구사항

### 5.1 회원 및 온보딩

- 사용자는 이메일, 소셜 로그인, Apple 로그인 중 하나로 가입할 수 있어야 한다.
- 최초 가입 시 성별 선택은 선택 사항이어야 하며, 추천 품질을 위해 스타일 선호 설문을 제공해야 한다.
- 사용자는 키, 체형, 선호 핏, 선호 색상, 피하고 싶은 스타일, 출근 복장 규정 등을 선택적으로 입력할 수 있어야 한다.
- 사용자는 기본 위치와 추천 알림 시간을 설정할 수 있어야 한다.

### 5.2 옷장 사진 촬영 및 업로드

- 사용자는 카메라로 새 사진을 촬영하거나 앨범에서 이미지를 업로드할 수 있어야 한다.
- 앱은 한 장의 사진에서 여러 개의 의류를 감지할 수 있어야 한다.
- 사용자는 자동 감지된 의류 영역을 추가, 삭제, 수정할 수 있어야 한다.
- 앱은 흐림, 어두움, 겹침, 낮은 해상도 등 품질 문제를 감지하고 재촬영을 안내해야 한다.
- 사용자는 의류 앞면, 뒷면, 디테일 사진을 추가로 등록할 수 있어야 한다.

### 5.3 의류 일러스트 변환

- 앱은 등록된 의류 이미지를 배경 제거 후 단일 의류 중심 이미지로 정제해야 한다.
- 앱은 실제 옷의 주요 형태, 색상, 패턴, 길이, 디테일을 유지한 일러스트 이미지를 생성해야 한다.
- 일러스트 스타일은 서비스 전체에서 일관된 톤을 유지해야 한다.
- 사용자는 원본 사진과 일러스트 이미지를 함께 확인할 수 있어야 한다.
- 사용자는 일러스트가 실제 옷과 다르면 재생성 요청을 할 수 있어야 한다.

### 5.4 의류 데이터화

앱은 각 의류에 대해 다음 속성을 자동 추출하거나 사용자가 수정할 수 있게 해야 한다.

- 카테고리: 상의, 하의, 아우터, 원피스, 신발, 가방, 액세서리 등
- 세부 유형: 셔츠, 니트, 후디, 데님, 슬랙스, 코트, 스니커즈 등
- 색상: 대표 색상, 보조 색상
- 패턴: 무지, 스트라이프, 체크, 플로럴, 그래픽 등
- 소재 추정: 면, 데님, 울, 니트, 가죽, 나일론 등
- 두께감: 얇음, 보통, 두꺼움
- 계절감: 봄, 여름, 가을, 겨울, 사계절
- 핏: 슬림, 레귤러, 오버핏, 와이드 등
- 포멀도: 캐주얼, 비즈니스 캐주얼, 포멀 등
- 스타일 태그: 미니멀, 스트릿, 클래식, 러블리, 빈티지 등
- 구매일, 브랜드, 가격, 사이즈, 메모
- 착용 횟수, 최근 착용일, 세탁 필요 여부

### 5.5 디지털 옷장 관리

- 사용자는 의류 목록을 카테고리, 색상, 계절, 착용 빈도, 최근 등록순으로 필터링할 수 있어야 한다.
- 사용자는 의류 상세 화면에서 이미지, 속성, 메모, 착용 기록을 확인할 수 있어야 한다.
- 사용자는 의류를 보관함, 세탁 중, 수선 중, 나눔/판매 예정 상태로 관리할 수 있어야 한다.
- 사용자는 계절별 보관 박스를 만들고 의류를 이동시킬 수 있어야 한다.
- 사용자는 잘 입지 않는 옷, 자주 입는 옷, 코디 활용도가 높은 옷을 분석할 수 있어야 한다.

### 5.6 코디 생성

- 앱은 최소 상의, 하의, 신발 조합을 기본 코디 단위로 생성해야 한다.
- 날씨나 상황에 따라 아우터, 가방, 액세서리를 추가 추천해야 한다.
- 앱은 색상 조화, 실루엣, 계절감, 포멀도, 소재감, 착용 이력을 고려해야 한다.
- 앱은 사용자가 이미 보유한 의류만으로 코디를 생성하는 모드를 제공해야 한다.
- 앱은 부족한 아이템을 포함한 제안 모드를 별도로 제공할 수 있어야 한다.
- 앱은 같은 옷이 지나치게 반복 추천되지 않도록 착용 빈도와 최근 착용일을 반영해야 한다.
- 앱은 추천 이유를 짧게 설명해야 한다.

### 5.7 자연어 스타일 요청

- 사용자는 텍스트로 원하는 분위기, 장소, 일정, 제한 조건을 입력할 수 있어야 한다.
- 앱은 사용자의 요청에서 다음 정보를 추출해야 한다.
  - 스타일 분위기
  - 착용 상황
  - 선호 또는 제외 색상
  - 포멀도
  - 편안함 우선 여부
  - 날씨 민감 조건
- 앱은 모호한 요청에 대해 필요한 경우 짧은 후속 질문을 할 수 있어야 한다.
- 앱은 추천 결과를 1개 이상의 코디 후보로 제공해야 한다.
- 사용자는 추천 결과에 대해 좋아요, 별로예요, 너무 더워요, 너무 추워요, 너무 튀어요 등의 피드백을 줄 수 있어야 한다.

### 5.8 트렌드 기반 추천

- 앱은 패션 트렌드 데이터를 주기적으로 수집하거나 외부 트렌드 API와 연동해야 한다.
- 트렌드 데이터는 지역, 계절, 성별 표현, 연령대, 플랫폼별 유행 차이를 구분할 수 있어야 한다.
- 앱은 트렌드 키워드를 서비스 내부 스타일 태그로 정규화해야 한다.
- 앱은 사용자의 보유 의류와 현재 트렌드 간의 매칭 가능성을 계산해야 한다.
- 앱은 유행 요소를 무조건 강제하지 않고 사용자 선호도와 착용 가능성을 우선해야 한다.
- 앱은 추천 화면에서 "트렌드 반영", "기본 스타일", "실험적인 스타일" 등의 강도 선택을 제공할 수 있어야 한다.
- 트렌드 데이터는 최신성이 중요하므로 최소 주 1회 이상 갱신되어야 한다.

### 5.9 날씨 기반 추천 및 알림

- 사용자는 기본 추천 시간을 설정할 수 있어야 한다.
- 사용자는 평일/주말 알림 시간을 다르게 설정할 수 있어야 한다.
- 앱은 사용자의 위치 기반 날씨를 조회해야 한다.
- 앱은 기온, 체감온도, 강수 확률, 강수 형태, 풍속, 습도, 자외선, 미세먼지를 고려할 수 있어야 한다.
- 비, 눈, 한파, 폭염, 미세먼지 등 특수 날씨에는 관련 아이템을 우선 추천해야 한다.
- 앱은 추천 시간 전에 코디 생성 작업을 완료해야 한다.
- 사용자는 푸시 알림을 통해 오늘의 추천 코디 상세 화면으로 진입할 수 있어야 한다.
- 날씨 API 장애 시 가장 최근 성공한 날씨 데이터와 일반 계절 규칙을 기반으로 대체 추천을 제공해야 한다.

### 5.10 일정 및 상황 연동

- 사용자는 수동으로 오늘의 일정 유형을 입력할 수 있어야 한다.
- 추후 캘린더 연동을 통해 출근, 운동, 데이트, 모임, 여행 등 상황을 자동 추정할 수 있어야 한다.
- 일정 연동은 사용자의 명시적 동의 후에만 활성화해야 한다.
- 일정 제목 등 민감 정보는 추천에 필요한 범위로만 처리해야 한다.

### 5.11 추천 결과 화면

- 추천 코디는 착장 이미지, 구성 아이템, 추천 이유, 날씨 정보, 스타일 태그를 함께 보여줘야 한다.
- 사용자는 추천 코디를 저장, 공유, 수정, 다시 추천받기 할 수 있어야 한다.
- 사용자는 특정 아이템을 고정한 상태로 나머지 아이템만 다시 추천받을 수 있어야 한다.
- 사용자는 마음에 들지 않는 아이템을 제외하고 대체 추천을 받을 수 있어야 한다.

### 5.12 스타일 기록

- 사용자는 실제 착용한 코디를 기록할 수 있어야 한다.
- 앱은 날짜별 착용 기록을 캘린더 형태로 제공해야 한다.
- 앱은 특정 옷의 착용 횟수와 코디 조합 이력을 보여줘야 한다.
- 앱은 사용자 피드백을 추천 모델 개선에 반영해야 한다.

## 6. 데이터 요구사항

### 6.1 주요 데이터 객체

- User: 회원, 프로필, 취향, 알림 설정
- ClosetItem: 의류 원본 이미지, 일러스트 이미지, 속성, 상태
- Outfit: 코디 조합, 추천 이유, 스타일 태그
- OutfitFeedback: 추천 결과에 대한 사용자 반응
- WeatherSnapshot: 추천 시점의 날씨 데이터
- TrendSignal: 수집된 트렌드 키워드와 가중치
- StylePreference: 사용자 취향과 제한 조건
- WearLog: 실제 착용 기록

### 6.2 데이터 보존

- 사용자는 언제든지 원본 사진, 일러스트, 의류 데이터를 삭제할 수 있어야 한다.
- 회원 탈퇴 시 개인 식별 정보와 사용자 생성 데이터는 삭제되어야 한다.
- 추천 품질 개선용 비식별 데이터 사용 여부는 별도 동의를 받아야 한다.
- 원본 사진은 사용자가 원하면 일러스트 생성 후 삭제하는 옵션을 제공해야 한다.

## 7. AI 및 모델 요구사항

### 7.1 이미지 인식

- 한 이미지에서 복수 의류를 감지할 수 있어야 한다.
- 의류 영역 분할, 배경 제거, 카테고리 분류, 속성 추출을 수행해야 한다.
- 낮은 신뢰도 결과는 사용자 검수를 요청해야 한다.
- 모델 결과에는 신뢰도 점수를 저장해야 한다.

### 7.2 일러스트 생성

- 원본 의류의 핵심 디테일을 유지해야 한다.
- 브랜드 로고나 저작권 요소가 있는 경우 과도한 복제를 피하고 식별 가능한 의류 특징 중심으로 표현해야 한다.
- 생성 결과는 사용자가 수정하거나 재생성할 수 있어야 한다.

### 7.3 추천 모델

- 추천 모델은 규칙 기반 필터와 AI 기반 랭킹을 함께 사용할 수 있어야 한다.
- 필수 필터: 날씨 부적합, 계절 부적합, 사용자가 제외한 아이템, 세탁 중 아이템
- 랭킹 요소: 사용자 취향, 트렌드 적합도, 색상 조화, 착용 빈도, 상황 적합도
- 추천 이유는 사용자가 이해할 수 있는 자연어로 생성해야 한다.
- 사용자 피드백을 반영하여 개인화 가중치를 조정해야 한다.

## 8. 외부 연동 요구사항

- 날씨 API: 위치 기반 현재 날씨와 시간대별 예보 조회
- 푸시 알림: iOS APNs, Android FCM
- 이미지 저장소: 원본 및 생성 이미지 저장
- 트렌드 데이터 소스: 패션 매거진, 소셜 키워드, 커머스 인기 검색어, 자체 큐레이션 데이터 중 하나 이상
- 지도/위치 서비스: 사용자 동의 기반 위치 조회
- 선택 연동: 캘린더, 커머스 상품 데이터, 소셜 공유

## 9. 비기능 요구사항

### 9.1 성능

- 단일 의류 등록 처리 결과는 일반적인 네트워크 환경에서 30초 이내 제공하는 것을 목표로 한다.
- 여러 의류가 포함된 사진 처리 시 진행 상태를 표시해야 한다.
- 오늘의 추천 코디는 알림 발송 시점 전에 미리 생성되어야 한다.
- 옷장 목록 화면은 1,000개 의류까지 부드럽게 탐색할 수 있어야 한다.

### 9.2 안정성

- 이미지 처리 실패 시 재시도 또는 수동 등록 경로를 제공해야 한다.
- 외부 API 장애 시 대체 로직과 사용자 안내를 제공해야 한다.
- 알림 발송 실패 이력을 기록하고 재시도 정책을 가져야 한다.

### 9.3 보안 및 개인정보

- 사용자 사진과 프로필 데이터는 전송 및 저장 시 암호화해야 한다.
- 위치 정보, 캘린더 정보, 사진 접근 권한은 명시적 동의 후 사용해야 한다.
- 권한 거부 상태에서도 수동 입력 기반 사용이 가능해야 한다.
- 내부 운영자는 원본 사진에 기본적으로 접근할 수 없어야 한다.
- 모델 학습 또는 품질 개선에 사용자 데이터를 사용할 경우 별도 동의와 철회 기능을 제공해야 한다.

### 9.4 접근성

- 주요 기능은 스크린 리더에서 이해 가능한 라벨을 가져야 한다.
- 색상만으로 상태를 구분하지 않아야 한다.
- 추천 결과의 핵심 정보는 텍스트로도 제공해야 한다.
- 작은 모바일 화면에서도 버튼과 텍스트가 겹치지 않아야 한다.

- 자연어 스타일 요청 화면

## 13. 정책 및 운영 요구사항

- 부적절한 이미지 업로드를 탐지하고 차단할 수 있어야 한다.
- 이미지 생성 결과가 실제 의류와 크게 다른 경우 신고 또는 재생성할 수 있어야 한다.
- 추천이 특정 체형, 성별, 외모 기준을 부정적으로 평가하지 않도록 해야 한다.
- 스타일 추천은 사용자의 자기표현을 돕는 방향이어야 하며 외모 비하성 표현을 피해야 한다.
- 관리자 도구에서 이미지 처리 실패율, 추천 실패율, 알림 발송 실패율을 확인할 수 있어야 한다.

## 14. 리스크 및 대응

- 사진 품질이 낮아 의류 인식이 실패할 수 있다.
  - 대응: 촬영 가이드, 품질 감지, 수동 보정 기능 제공
- 일러스트가 실제 옷과 다르게 생성될 수 있다.
  - 대응: 원본 비교, 재생성, 사용자 수정 플로우 제공
- 최신 트렌드 데이터가 부정확하거나 편향될 수 있다.
  - 대응: 복수 데이터 소스, 큐레이션 검수, 사용자 선호 우선 랭킹 적용
- 날씨 API 장애로 아침 추천이 실패할 수 있다.
  - 대응: 캐시 데이터, 계절 기반 대체 추천, 장애 알림 로그
- 개인정보 민감도가 높다.
  - 대응: 명시적 동의, 암호화, 삭제권, 데이터 최소 수집 원칙 적용

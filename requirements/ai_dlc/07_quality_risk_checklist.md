# 07. Quality and Risk Checklist

## 1. 테스트 전략

### Unit Test

- 의류 속성 정규화
- 날씨 기반 필터
- 코디 후보 생성 규칙
- 추천 점수 계산
- 알림 시간 계산
- 사용자 권한 상태 처리

### Integration Test

- 이미지 업로드 후 AI 작업 생성
- AI 작업 완료 후 ClosetItem 저장
- 자연어 추천 요청 후 Outfit 생성
- 날씨 API 실패 시 캐시/대체 추천
- 푸시 발송 작업 생성
- 계정 삭제 시 이미지와 메타데이터 삭제

### E2E Test

- 가입 -> 온보딩 -> 의류 등록 -> 추천 요청 -> 피드백
- 알림 설정 -> 예약 추천 생성 -> 푸시 진입
- AI 분석 실패 -> 수동 수정 -> 저장

### AI Evaluation

- 샘플 의류 이미지 세트로 카테고리/색상 정확도 측정
- 일러스트 결과의 사용자 재생성률 측정
- 추천 결과에 대한 긍정/부정 피드백 사유 분석
- 날씨 부적합 피드백률 추적

## 2. 비기능 체크리스트

### Performance

- 단일 의류 처리 목표: 30초 이내
- 추천 생성 목표: 10초 이내
- 옷장 목록 1,000개까지 스크롤 성능 확인
- 이미지 업로드 진행률 표시
- 장시간 AI 작업은 백그라운드 처리

### Reliability

- AI 작업 재시도 정책
- 외부 API timeout과 fallback
- 실패 작업 상태 저장
- 푸시 발송 실패 재시도
- 날씨 데이터 캐싱

### Security

- 모든 API 인증 적용
- 사용자별 데이터 접근 제어
- pre-signed upload URL 만료 시간 제한
- object storage key 예측 불가 처리
- 원본 이미지 내부 접근 제한
- 민감 로그 마스킹

### Privacy

- 사진, 위치, 알림 권한에 대한 명시적 동의
- 원본 이미지 삭제 옵션
- 회원 탈퇴 시 데이터 삭제
- 모델 개선용 데이터 사용 별도 동의
- 데이터 사용 동의 철회 가능

### Accessibility

- 이미지 버튼에 접근성 라벨 제공
- 추천 이유를 텍스트로 제공
- 색상 외 상태 표시 제공
- 작은 화면에서 텍스트 겹침 방지
- 터치 영역 최소 크기 준수

## 3. 주요 리스크

| Risk | Impact | Probability | Mitigation |
| --- | --- | --- | --- |
| 의류 인식 정확도 부족 | 높음 | 중간 | 단일 의류 MVP, 사용자 검수, 신뢰도 표시 |
| 일러스트가 원본과 다름 | 높음 | 중간 | 재생성, 원본 비교, 스타일 고정 프롬프트 |
| 이미지 생성 비용 증가 | 중간 | 중간 | 캐싱, 지연 생성, 썸네일 품질 단계화 |
| 날씨 API 장애 | 중간 | 중간 | 캐시, 계절 규칙 fallback |
| 추천이 취향과 맞지 않음 | 높음 | 높음 | 피드백 루프, 보수적 기본 추천, 제외/고정 기능 |
| 개인정보 우려 | 높음 | 중간 | 원본 삭제 옵션, 명시적 동의, 접근 제한 |
| 트렌드 데이터 편향 | 중간 | 중간 | 다중 소스, 운영 검수, 사용자 취향 우선 |
| Expo SDK 52 전이 의존성 보안 경고 | 중간 | 중간 | 개발 중 lockfile 고정, 외부 입력을 빌드 도구에 전달하지 않음, 베타 전 지원 SDK로 업그레이드 |

## 4. 운영 지표

- 이미지 업로드 성공률
- AI 분석 성공률
- 일러스트 생성 성공률
- 평균 의류 등록 처리 시간
- 추천 생성 성공률
- 푸시 발송 성공률
- 푸시 클릭률
- 추천 저장률
- 추천 긍정 피드백률
- 데이터 삭제 요청 처리 시간

## 5. 출시 전 게이트

### Product Gate

- 핵심 플로우가 5분 안에 완료된다.
- 첫 추천 결과가 사용자 보유 의류만으로 생성된다.
- 추천 이유가 사용자가 이해할 수 있는 문장으로 표시된다.

### AI Gate

- 카테고리 자동 추출 정확도가 내부 샘플에서 목표 기준을 만족한다.
- 대표 색상 수정률이 과도하게 높지 않다.
- 일러스트 재생성률이 허용 기준 이하이다.
- 날씨 부적합 피드백률이 허용 기준 이하이다.

### Engineering Gate

- P0 API에 인증과 권한 검사가 적용된다.
- 주요 작업 실패 상태가 사용자에게 표시된다.
- 개인정보 삭제 플로우가 검증된다.
- 알림 스케줄러가 시간대별로 테스트된다.

## 6. 결정 필요 사항

- 앱 스택: React Native Expo 또는 Flutter
- 백엔드 스택: FastAPI 또는 Node.js 계열
- 인증 제공자: 자체, Firebase Auth, Supabase Auth 등
- 이미지 저장소: AWS S3, Cloudflare R2, Supabase Storage 등
- 날씨 API: OpenWeather, WeatherAPI, 기상청 API 등
- AI API: vision, image generation, LLM 제공자
- 트렌드 데이터 운영 방식: 수동 큐레이션, 외부 API, 크롤링, 커머스 검색어 제휴
- MVP 디자인 수준: 와이어프레임 우선 또는 바로 고충실도 UI
